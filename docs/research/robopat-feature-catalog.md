# ロボパットDX 機能カタログ（競合分析・一次インプット）

> 競合RPA製品 **ロボパットDX（Robo-Pat DX）** の基礎編トレーニング教材から抽出した
> 機能・コマンドの網羅リスト。Hermes へ機能を追加する際の参照元（インプット）として用いる。
>
> - **出典**: `docs/references/基礎編（使用ツール・配布資料）/` 配下の配布PDF6本＋ロボット定義 `.bwn`（ローカル限定）。
> - **抽出方法**: 各PDFを画像として精読し機能を抽出（自動分析、計120項目＋ショートカット33件）。
> - **著作権**: 本ファイルは原資料の逐語転記ではなく、機能を自分の言葉で要約した二次分析。数値・画面文言の引用は最小限。提供元は株式会社FCE（Copyright FCE 表記）。
> - **重要な前提差**: ロボパットは **画像認識型（テンプレートマッチング）＋キーストローク送出** が中核で **Windows専用**。Hermes は AX/DOM セレクタ中心・macOS 先行。アーキテクチャが異なるため「同じ機能」でも実装方式は読み替えが要る（→ `robopat-vs-hermes-gap.md`）。

## 0. 製品アーキテクチャ（機能を読む前提）

| 項目 | 内容 |
|---|---|
| 自動化方式 | 画面キャプチャ画像を**テンプレートマッチング**で探索 → マウス/キーボード操作。補助的に**座標指定**も可。 |
| 対象 | 任意のアプリ/ファイル/フォルダ、ブラウザ（Web記録）、Excel（特化コマンド）。画面に映るものなら種別を問わず操作可。 |
| 動作環境 | Windows 8.1 / 10 / 11（**Windows専用**）。 |
| 実行モード | ①デスクトップ（画像認識）②Web（WebDriver経由）③Excel特化。 |
| 内部モデル | ロボット定義 `.bwn` = **Javaシリアライズされた HashMap**（内部コードネーム **Brownie** / `com.asirrera.brownie`）。`projectName` + `scriptData`（`commandData` リスト）。 |
| コマンド構造 | 各コマンド = `Arguments`（引数）＋ `FindOptionModel`（画像探索設定）＋ `RetryIf`（リトライ条件）＋ `INDENT`（ブロックのネスト深さ）＋ `COMMENT`（行コメント）＋ `LINE_NUMBER`。 |
| 既定パラメータ | `DefaultMatch=0.8`（既定一致率）/ `SearchTimeout=30000`ms（画像探索タイムアウト）/ `WaitTimeForCommon=300`（共通待機）/ `MouseMoveSpeed` / `IsVisibleForMouseTrajectory`（マウス軌跡表示）/ `IsAutoReduceMatchRateBefore1420`（一致率の自動緩和・版依存）。 |
| 安定稼働の思想 | 「**待機**」を多用してタイミングを合わせる設計。1コマンドごとの既定実行間隔 0.5 秒。 |

### 画像認識を安定させるための環境前提（ロボパット特有の運用負荷）
画像認識型ゆえに、稼働PCの見た目を固定する初期設定が必須。これは **画像認識方式の弱点を運用で補う** もので、AX/DOM方式の Hermes には基本的に不要な領域（重要な差別化点）。

- **背景色を単色化**（壁紙が写真だと対象を誤判別）。
- **拡大率 100%**（100〜300% 設定可。変えると一致率/解像度が変わり、**端末ごとにスクリプト作り直し**が必要）。
- **パフォーマンスオプション**：視覚効果オフ＋「スクリーンフォントの縁を滑らかにする」のみオン → 文字判別/文字読み取り（OCR系）の精度向上。
- リモートデスクトップ/マルチPCでは画像コマンド入りスクリプトの共有に制約。

## 3. サポート・学習エコシステム（非機能的な競合の強み）

製品機能ではないが、ロボパットの導入定着力を支える付帯サービス。Hermes（OSS・source-only）とは性質が異なるが、競合分析として記録。

- **ヘルプデスク**（Web、1社1アカウント、メール問い合わせは2営業日返信）
- **動画チャンネル**（SmartBoarding 上で配信、基本操作〜便利機能）
- **Web家庭教師**（ZOOM等で1コマ30分・予約制の個別相談）
- **操作勉強会**（基礎編/応用編、オンライン）
- **ロボパットマスター認定プログラム**（ベンダー資格、研修＋認定）
- **各種セミナー**（自動化洗い出し＋社内体制構築、経営幹部・管理職向け）
- **フォローアップ教室**（oVice/ZOOM、週1・予約制）
- **ヘルプサイト**（FAQカテゴリ: 環境設定/操作方法/役立ち事業集/コマンド設定例/トラブル/プラグイン、キーワード検索）

## 4. ロボット定義 `.bwn` の内部モデル（実サンプルより）

サンプル「郵便番号並べ替え.bwn」を展開・解析した実測。Hermes IR を設計する際の比較対象。

- 形式: Java直列化 HashMap（`projectName` / `scriptData{ MergeInfoData, tabTitle:"main", commandData:[...] }` / `projectProperties{...}`）。
- 各コマンドのフィールド: `Arguments` ＋ `FindOptionModel`（画像探索）＋ `RetryIf`（リトライ）＋ `LINE_NUMBER` ＋ `INDENT`（ネスト深さ）＋ `TAB_TITLE` ＋ `COMMENT`（行コメント）。
- サンプルの中身: Excelのキーボードショートカット送出列（Ctrl+Home → Shift+Ctrl+Right/Down で範囲選択 → Shift+Ctrl+L フィルタ → Alt+Down → S 昇順）。各行に日本語コメント付き。
- プロジェクト設定: `BwnVersionLastUpdatedScript=1.5.0.7` / `DefaultMatch=0.8` / `SearchTimeout=30000` / `WaitTimeForCommon=300` / `MouseMoveSpeed=0` / `IsVisibleForMouseTrajectory=true` / `IsSendKeysJava` / `IsConcatenateString` / `ProductUuid`。

**設計示唆**: ロボパットのコマンドは「対象探索(FindOption) + 引数 + リトライ + ネスト + コメント」が1単位。Hermes IR の `Step{ target(候補配列) + params + retry + children + label }` とほぼ同型で、**Hermes IR の方が target を候補配列（AX/DOM/画像/座標のフォールバック）で持つ点が構造的に強い**。

## 5. ひと言まとめ（Hermes視点）

- ロボパットの**中核 = 画像認識（テンプレートマッチ）＋一致率＋多彩な待機＋キーストローク送出**。これらを「画像が消えるまで待つ／一致率調整／座標フォールバック」まで丁寧に作り込んでいるのが強み。
- 一方で **Windows専用・環境固定（背景色/拡大率/パフォーマンス）が必須・フロー制御がGoTo+タブ** と、構造面は素朴。
- Hermes は AX/DOM 中心・構造化制御・AI判定層・Vault と**設計が上**だが、**画像認識/OCRの実行系・マウス操作の網羅・各種待機・Excel・通知/スケジュール**が手薄。具体的な突き合わせと追加提案は **`robopat-vs-hermes-gap.md`** を参照。
