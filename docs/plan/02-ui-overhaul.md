# UI 刷新計画

> 要件（ユーザー）: 「ちゃんとアプリとして使いやすいUIに。いらない説明文・いらないパラメータも全部消す。」
> 本書は刷新の計画。**実装はしない。**

---

## 結論（先に）

現状UIは「機能するが、技術がそのまま露出した開発者向けの見た目」。これを **3つの原則**で作り変える:

1. **段階的開示（Progressive Disclosure）** — 既定では「人が読む情報」だけ。生JSON・ID・低レベル設定は「詳細/上級モード」に隠す。
2. **スマートデフォルト** — 不要なパラメータは画面から消し、賢い既定値で埋める。露出するのは「意味のある選択」だけ。
3. **デザインシステム化** — 手書きCSSをやめ、一貫したトークン（余白・タイポ・色）と部品（PLAN.md が元々意図した Radix/shadcn + dnd-kit）で「アプリらしい」見た目に。

---

## 現状診断（実測）

| 項目 | 現状 | 問題 |
|---|---|---|
| 構成 | `App.tsx` が**1,762行の単一ファイル**に全UI、`store.ts` 796行 | 巨大モノリス。保守・改善が困難 |
| スタイル | 手書き `styles.css` 518行・103セレクタ、フレームワーク無し | デザインシステム不在。一貫性・磨き込みが効かない |
| 構造 | 3ペイン（フロー一覧 `FlowSidebar` ｜ タイムライン `Timeline/StepNode` ｜ インスペクタ `Inspector/StepEditor`）＋設定 `AppSettingsPanel`＋Vault `VaultPanel` | 骨格は妥当。中身の露出が過剰 |
| パラメータ露出 | Inspector が**生JSONの target を表示**（`<pre>{JSON.stringify(step.target)}</pre>`）、step ID 表示（`#xxxxxx`）、timeout/retry等の低レベル項目 | 技術内部がそのまま見える＝「使いにくい」の主因 |
| 用語 | `domcontentloaded (DOM のみ)` / `networkidle (通信が止まる)` / `AX 要素の出現` / `カスタム条件式` 等が生のまま | RPA/Web内部用語がユーザーに漏れている |
| メニュー | 挿入メニューに wait_for が**9種フラット**に並ぶ | 低レベルすぎる選択肢の洪水 |
| 説明文 | 式言語ヘルプ等がインライン（`var.x, secrets.x ... contains/startsWith`）、空状態の冗長文 | 画面が説明で埋まる |

> 既に良い点: 主要ラベルは日本語化済み（「ページのロード完了で次へ」「画面が落ち着いたら次へ」等）。この"言い換えの良さ"を全体へ徹底する。

## 「消す」具体リスト（不要な説明文・パラメータ）

刷新で既定表示から除去 or 移設するもの:

- **生JSON の target 表示**（`Inspector`）→ 既定は「対象: ボタン〈ログイン〉」のような人間可読サマリ。生JSONは「詳細」展開時のみ。
- **step ID 表示**（`#a1b2c3`）→ 既定は非表示。デバッグ時のみ。
- **wait_for 9種のフラット露出** → 「待つ」1メニュー＋文脈に応じた数個に集約。残りは「高度な待機」配下。
- **インライン式ヘルプ**（`var.x, secrets.x ...`）→ 「?」ツールチップ/ヘルプポップオーバーへ。
- **timeout/retry/onError 等の低レベル項目** → 既定は隠し、スマートデフォルト。「詳細設定」展開でのみ編集可。
- **冗長な空状態文** → 1行＋主要アクションボタンに置換（例: 「フローを選択」→ 新規/録画ボタンを前面に）。
- **技術用語の生表記**（domcontentloaded/networkidle/AX/expr）→ UIから消し、必要なら補足ツールチップへ。

## 刷新の方針（領域別）

### 1. デザインシステム導入
- トークン化テーマ（色・余白・角丸・影・タイポのスケール）。ダーク基調＋アクセント。
- 部品: Radix プリミティブ + shadcn/ui 流のコンポーネント、`dnd-kit` でタイムラインのDnD。手書きCSSを段階的に置換。
- 「Webページ」ではなく「アプリ」の質感（適切なクローム、密度、アイコン、状態色）。

### 2. App.tsx の分解
- 1,762行を機能単位に分割: `FlowSidebar/` `Timeline/`（`StepNode`,`InsertHandle`,`BranchSection`）`Inspector/`（`StepEditor`,`WaitEditor`,`WaitForEditor`）`SettingsPanel/` `VaultPanel/` `RunLog/` `modals/`（Prompt/Confirm）。
- これ自体は挙動を変えないリファクタ。刷新の前提として先に行う。

### 3. インスペクタの再設計（パラメータ削減の核）
- **2段階**: 「基本」（対象＋意味のある1〜2項目）と「詳細」（折りたたみ: timeout/retry/onError/生target/アサーション）。
- 対象(target)は人間可読サマリ＋「対象を取り直す/録画し直す」アクション。候補セレクタ配列の生表示はしない。
- ステップ種別ごとに、本当に必要なフィールドだけを出す（不要パラメータはスキーマ既定で補完）。

### 4. 用語と説明の整理
- 既存の良い言い換えを全画面へ徹底。生の技術語はツールチップに退避。
- 説明文は「画面に常駐させない」。必要時にだけ「?」で出す。

### 5. 主要動線（使いやすさの本体）
- **録画 / 実行 / 編集** の3アクションを明快に。録画ボタン・実行ボタンを常に手の届く位置に。
- 実行中の状態（進行・現在ステップ・成否）と Run ログの視認性を上げる。
- キーボードショートカット、メニューバー、ネイティブ感のあるウィンドウ統合で「Webアプリ感」を払拭。

## 段階（順序）

| # | フェーズ | 内容 | 検収 |
|---|---|---|---|
| 1 | デザインシステム導入 | トークン/テーマ/基本部品、手書きCSSの置換着手 | 主要画面が新トークンで描画され一貫 |
| 2 | App.tsx 分解 | 機能単位コンポーネント化（挙動不変） | 既存機能が回帰なく動く |
| 3 | インスペクタ再設計 | 基本/詳細の2段化、生JSON/ID/低レベル項目を隠す、スマートデフォルト | 既定画面から技術露出が消える |
| 4 | 用語・説明整理 | 生技術語の退避、インライン説明のツールチップ化、空状態簡潔化 | 不要な説明文がゼロ |
| 5 | 動線磨き上げ | 録画/実行/編集の主要動線、実行状態・ログUX | 「アプリとして使いやすい」体感 |

---

## 注意

- これは UI の**計画**。実装は技術スタック地ならし（`00`/`01` の補強）や機能追加と並行/別途で進める。
- UI言語は日本語のまま（PLAN.md 準拠、i18next 枠は維持）。
- 「使いやすさ」は主観なので、フェーズ5の前に主要画面のモックで方向性を一度すり合わせるのが望ましい。
